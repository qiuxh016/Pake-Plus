## 4. 离线缓存（1:10-1:35）

| 画面                                           | 画外音                                                                                                                                                                                                                                                                                          |
| ---------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 浏览页面，右下角弹出绿色 toast "⚡ from cache" | 离线缓存模块在 HTTP 层透明代理——`cache.js` 替换 `window.fetch`，每个 GET 请求通过 `cache_fetch` IPC 先查 Rust 后端的本地缓存索引（`CacheIndex`，以 URL 哈希为文件名的磁盘存储）。命中直接返回缓存数据（延迟从百毫秒降至毫秒），未命中通过 `tauri_plugin_http::reqwest` 发起请求并异步写入缓存。 |
| 设置面板 → Cache 标签，展示实时统计            | 缓存统计通过 `cache_stats_json` IPC 从运行中引擎实时查询，包含文件数、已用空间、命中率（近1小时）和命中/未命中次数。缓存上限通过滑块调整（50-1000MB），采用 LRU 淘汰算法——超过上限时按最后访问时间升序删除，每次至少淘汰 10MB。                                                                 |
| 断网演示                                       | 监听 `online/offline` 事件自动切换离线模式：顶部黄色横幅 + 左下角 OFFLINE 徽章出现，已缓存页面正常访问，未缓存页面显示友好离线提示页。                                                                                                                                                          |

## 技术关键词（录制时可穿插提及）

- **后端**：Rust + Tauri v2 + `tauri-plugin-http` + `rusqlite` + `zip` + `sysinfo` + `rfd` + `arboard`
- **前端**：Vanilla JS（无框架）+ DOM API + IIFE 注入
- **IPC**：`#[tauri::command]` + `invoke()`，30+ 命令，ACL 权限统一管理
- **测试**：Rust 67 项 + JS 297 项 + 整合 85 项，全部通过

## 录制要点

1. 先完整走一遍流程再录，避免卡顿
2. Windows 缩放调到 150%，关键区域放大
3. 关闭无关窗口，保持桌面整洁
4. 广告拦截和离线缓存要有"开/关"前后对比
5. 时间控制：片头和对比可各压缩 5 秒
